# Prompt: Test generation

Add the missing tests for a target area.

Please:

1. Read the implementation and the existing tests.
2. List the missing cases — authorization failures, error paths, and domain edge cases.
3. Add the tests before changing production code, so a real gap fails first.
4. Use the project's existing test style and fixtures; avoid over-mocking.
5. Run the relevant suite and report the result and the command used.

Prioritize authorization-sensitive endpoints and state transitions.
